# proto-spec-generator

原型设计规格文档生成 Skill。将需求文档中的功能描述转化为逐页标准化 MD 规格文档，按端→模块→页面四级结构输出，作为 HTML 原型开发的前置输入。

## 适用场景

- 招投标总表 / PRD / 业务流程文档 → 逐页原型设计规格
- B 端管理后台和 C 端 H5 / 小程序项目
- 需要交付给研发的结构化页面规格

## 工作流程

0. **记录需求来源** → 确保规格可溯源
1. **端侧确认** → 门户 / 管理后台 / 个人中心等端侧划分
2. **UI 基线确认** → 从设计库加载对应端态配色、字号、组件参数
3. **需求逐句拆解** → 每条功能标注来源，推断性功能明确标记
4. **模块对齐** → 一级模块名严格对齐需求文档
5. **业务流程图** → 明确每页在流程中的位置
6. **页面划分** → 逐模块确定页面数量与功能承载关系
7. **已有能力复用** → 避免重复设计平台已有功能
8. **产出架构清单** → 端/模块/页面/功能四级映射表

## 输出

标准化 MD 文件，固定章节结构：功能概述 → 权限说明 → 页面结构 → 页面内容（逐区域） → 交互规则。

## 下游

规格文档可直接输入 [prototype-html-pin](https://github.com/wsdlp46/prototype-html-pin) 生成可交互 HTML 原型。

> 完整产品工作流建议搭配 [pm-assistant-lrw](https://github.com/wsdlp46/pm-assistant-lrw) Expert，一站式覆盖 PRD → 方案 → 规格 → 原型。
